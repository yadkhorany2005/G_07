This document shows the following use case diagram for Mental health disorders website, it connects user with the psychiatric services and mental health resources, the website contains detailed platforms for mental health management, professional connection, personalized support




 
Actors:
Primary actor: User, a person who looks for a mental health support or information or mental services
Secondary actor: psychiatrist, a specialist and licensed mental health professional who offers clinical services through the website


Use Cases:
Mandatory:(pink )
1.log in /authentication: (actor: user) user can use the website by creating an account and having a good secure to protect users’ privacy, user must login but can change the display info into anonymous.
Main Flow:
1.	User navigates to system homepage 
2.	Chooses login, signup, or anonymous access 
3.	System grants appropriate access 
4.	User proceeds to features
   
Business: anonymous mode can get limited access


2.get guidance: (actor: user) user gets general guidance about mental health topics like (autism, depression…etc) when they need professional help.

Main Flow:
1.	User selects "Get Guidance" 
2.	Browses or searches topics 
3.	Reviews and saves guidance materials
3.take questionnaires: (actor: user) user complete mental health assessments questionnaire to evaluate their effects and conditions severity.

Main Flow:
1.	1.User selects available questionnaire 
2.	2.Completes assessment 
3.	3.System calculates and displays results 

Business: critical scores should get immediate support options.

4.information about disorders: (actor: user) user gets comprehensive information about various mental health disorders like (treatments, management strategies)

Main Flow:
1.	User searches or browses disorder categories 
2.	Selects specific disorder 
3.	Reviews evidence-based information 
4.	Can save or share content

Nice to have(blue):

5.Track Progress: (actor: user) users can send feedback of mental health journey during that time through visualizations and historical data of their assessments and activities

Main Flow:
1.	User selects "Track Progress" 
2.	System displays charts, graphs, and trends 
3.	User reviews patterns and receives insights
6.connect with psychiatrist: (actor user) users can schedule appointments with licensed specialists and communicate with them.

Main Flow:
1.	User browses psychiatrist profiles 
2.	Filters by specialty or availability 
3.	Requests appointment 
4.	Psychiatrist confirms 
5.	Connection established via platform
7. Tailored Support: the website system shows the personalized support recommendations based on user assessments and progress.

Main Flow:
1.	System analyzes user data 
2.	Generates personalized recommendations 
3.	User accesses customized coping strategies and resources
8. User Account Management: (actor user) user can manage their account setting, privacy preferences, and personal information.

Main Flow:
1.	User accesses account settings 
2.	Updates profile, privacy, or security settings 
3.	System saves changes

Superfluous(green):

9. AI Companions:(actor user) users can interact with powered Ai for emotional support and daily check-ins.

Main Flow:
1.	User initiates conversation 
2.	Shares feelings or concerns 
3.	AI provides empathetic responses and suggestions 
4.	Crisis situations escalated to professionals

10. Location Guidance: (actor user) Users find nearby mental health resources, facilities, and support groups.

Main Flow:
1.	User provides or enters location
2.	System searches nearby resources
3.	Displays map with clinics, crisis centers, and services
4.	User views details and directions

11. Online Therapy Sessions: (actor user) Users participate in secure video-based therapy sessions with licensed professionals.

Main Flow:
1.	Both parties join at scheduled time
2.	System establishes secure video connection
3.	Session conducted and documented
4.	Follow-up scheduled if needed
